;=============================================
; Known account gets a reset link and a generic confirmation.
;=============================================
GIVEN a signed-out user with an existing account.
WHEN they submit their email at the Forgot password page.
THEN a single-use time-limited reset link is sent to that email.
THEN a generic check-your-email confirmation is shown.

;=============================================
; Unknown email gets the same generic confirmation and no email is sent.
;=============================================
GIVEN an email address with no account.
WHEN it is submitted at the Forgot password page.
THEN no email is sent.
THEN the same generic check-your-email confirmation is shown.
THEN there is no indication that the account is missing.

;=============================================
; Valid reset link lets the user set a new password.
;=============================================
GIVEN a valid unused reset link.
WHEN the user opens the link and sets a new password meeting the policy.
THEN the password is updated.
THEN the user can sign in with the new password.

;=============================================
; Successful reset signs out all other active sessions.
;=============================================
GIVEN a user who has just completed a password reset.
WHEN the reset succeeds.
THEN all other active sessions for that account are signed out.

;=============================================
; Used or expired reset link is rejected with a message and a way forward.
;=============================================
GIVEN a reset link that has already been used or has expired.
WHEN the user opens the link.
THEN it is rejected with a clear message.
THEN the user is offered an option to request a new link.

;=============================================
; New password that violates the policy is rejected without changing the password.
;=============================================
GIVEN a new password that violates the policy such as fewer than twelve characters.
WHEN the user submits it on the reset page.
THEN it is rejected with guidance.
THEN the password is unchanged.
